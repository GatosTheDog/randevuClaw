`ALTER TABLE services ENABLE ROW LEVEL SECURITY;`
`ALTER TABLE business_hours ENABLE ROW LEVEL SECURITY;`
`ALTER TABLE client_business_relationships ENABLE ROW LEVEL SECURITY;`
`ALTER TABLE conversation_turns ENABLE ROW LEVEL SECURITY;`
`ALTER TABLE businesses ENABLE ROW LEVEL SECURITY;`

For messages, bookings, services, business_hours, client_business_relationships, conversation_turns: policy checks `business_id = current_setting('app.current_business_id', true)::INTEGER`. For businesses: checks `id = current_setting('app.current_business_id', true)::INTEGER`. Use a single `FOR ALL` policy per table (combines USING and WITH CHECK in one statement for brevity).

Section 4 — Grant permissions (D-11). For business-scoped tables (messages, bookings, services, business_hours, client_business_relationships, conversation_turns): `GRANT SELECT, INSERT, UPDATE, DELETE ON <table> TO randevuclaw_app;`. Wrap each ENABLE ROW LEVEL SECURITY in `IF NOT EXISTS (SELECT FROM pg_policies WHERE tablename = ...)` or use `ALTER TABLE ... ENABLE ROW LEVEL SECURITY` which is idempotent. Wrap each CREATE POLICY in a `DO $$ BEGIN IF NOT EXISTS (SELECT FROM pg_policies WHERE policyname = '...' AND tablename = '...') THEN CREATE POLICY ...; END IF; END $$;` block.
