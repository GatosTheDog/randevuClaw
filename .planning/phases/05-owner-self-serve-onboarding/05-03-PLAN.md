    TIME_REGEX constant (not exported): /^([01]\d|2[0-3]):[0-5]\d$/.

    Helper: parseCollectedData(raw: string | null): CollectedData — JSON.parse(raw ?? '{}') with try/catch fallback to {}.
    Helper: serializeCollectedData(data: CollectedData): string — JSON.stringify(data).

    Export nine handler functions (all async, all called from router.ts):

    handleNameStep(session, business, ownerTelegramId, text): Validate text.trim() is non-empty and <= 100 chars. Generate slug from trimmed text using generateSlug (import from src/database/seed.ts). Fetch existing slugs (db.select({slug: businesses.slug}).from(businesses)) to pass to generateSlug for collision avoidance. Update businesses SET name = trimmedName, slug = generatedSlug WHERE id = business.id (use db.update directly — admin). Call updateOnboardingStep(session.id, 'hours_0_query', null). Send: 'Είστε ανοιχτά την Κυριακή; (ναι/όχι)'.

    handleHoursQueryStep(day, session, business, ownerTelegramId, text): Normalize text: text.trim().toLowerCase(). If includes 'ναι' or 'yes': advance step to `hours_${day}_open`, send: 'Ώρα έναρξης {GREEK_DAY_NAMES[day]} (π.χ. 09:00):'. If includes 'όχι' or 'no': INSERT business_hours row with { businessId: business.id, dayOfWeek: day, openTime: '00:00', closeTime: '00:00', isClosed: true } using db.insert(businessHours).values(...).onConflictDoNothing(). Advance step to next day's query OR 'svc_name' if day == 6. If next step is 'svc_name', also clear collectedData. If response not recognized, re-send the same question without advancing.

    handleHoursOpenStep(day, session, business, ownerTelegramId, text): Validate TIME_REGEX on text.trim(). On failure: send 'Μη έγκυρη ώρα. Χρησιμοποιήστε μορφή ΩΩ:ΛΛ (π.χ. 09:00):', return without advancing. On success: save to collectedData.currentDayOpenTime, call updateOnboardingStep(session.id, `hours_${day}_close`, serializedCollectedData). Send: 'Ώρα λήξης {GREEK_DAY_NAMES[day]} (π.χ. 18:00):'.

    handleHoursCloseStep(day, session, business, ownerTelegramId, text): Validate TIME_REGEX on text.trim(). On failure: same error message, return. On success: read collectedData.currentDayOpenTime. INSERT business_hours row with { businessId: business.id, dayOfWeek: day, openTime: collectedData.currentDayOpenTime, closeTime: text.trim(), isClosed: false } using db.insert(businessHours).values(...).onConflictDoNothing(). Clear currentDayOpenTime from collectedData. Advance step to next day's query OR 'svc_name' if day == 6. If next step is 'svc_name': clear collectedData.currentService. Send appropriate prompt: either 'Είστε ανοιχτά την {nextDayName};' for next day or 'Ωραία! Προσθέστε μια υπηρεσία.\nΌνομα υπηρεσίας:' for services.

    handleSvcNameStep(session, business, ownerTelegramId, text): Validate non-empty, <= 100 chars. Clear collectedData.currentService and set currentService.name = text.trim(). Call updateOnboardingStep(session.id, 'svc_price', serialized). Send: 'Τιμή σε λεπτά ευρώ (π.χ. 2000 = 20€):'.

    handleSvcPriceStep(session, business, ownerTelegramId, text): Parse text as integer. Validate: parsed integer > 0. On failure: send 'Μη έγκυρη τιμή. Εισάγετε θετικό ακέραιο αριθμό (π.χ. 2000):'. On success: set collectedData.currentService.price = parsed. Advance to 'svc_duration'. Send: 'Διάρκεια σε λεπτά (π.χ. 60):'.

    handleSvcDurationStep(session, business, ownerTelegramId, text): Parse text as integer. Validate > 0. On failure: send 'Μη έγκυρη διάρκεια. Εισάγετε θετικό ακέραιο (π.χ. 60):'. On success: INSERT into services { businessId: business.id, name: collectedData.currentService.name, price: collectedData.currentService.price, durationMin: parsed } using db.insert(services).values(...). Clear collectedData.currentService. Advance step to 'svc_more'. Send: 'Υπηρεσία αποθηκεύτηκε! Θέλετε να προσθέσετε άλλη; (ναι/όχι)'.

    handleSvcMoreStep(session, business, ownerTelegramId, text): Normalize to lowercase. If 'ναι' or 'yes': set collectedData.currentService = {} (per Pitfall 6 — clear stale partial data). Advance to 'svc_name'. Send: 'Όνομα νέας υπηρεσίας:'. If 'όχι' or 'no': call handleActivate. If unrecognized: re-send the question.

    handleActivate(session, business, ownerTelegramId): Generate new webhookId = crypto.randomUUID(), webhookSecret = crypto.randomBytes(32).toString('hex'). Call unregisterBotWebhook(business.botToken!) first (always, per STATE.md blocker). Then call registerBotWebhook(business.botToken!, `${config.webhookBaseUrl}/webhooks/telegram/${webhookId}`, webhookSecret). Call activateBusiness(business.id, webhookId, webhookSecret). Call updateOnboardingStep(session.id, 'done', null). Send: 'Η επιχείρησή σας είναι ενεργή! Οι πελάτες μπορούν τώρα να κάνουν κράτηση μέσω του bot σας.'.

    Import config from src/config.ts for webhookBaseUrl in handleActivate.

| owner input → step handlers | All owner-supplied text passes through step handlers before reaching DB or Telegram API |
| step handlers → Telegram API | handleActivate calls registerBotWebhook with owner's botToken; token must not be logged |

## STRIDE Threat Register

| Threat ID | Category | Component | Severity | Disposition | Mitigation Plan |
|-----------|----------|-----------|----------|-------------|-----------------|
| T-05-06 | Tampering | Malformed HH:MM time input causing downstream slot-calculation errors | medium | mitigate | TIME_REGEX = /^([01]\d|2[0-3]):[0-5]\d$/ validates before DB write; invalid input re-prompts without advancing step |
| T-05-07 | Tampering | Business name > 100 chars or with special chars | low | mitigate | handleNameStep validates len <= 100; generateSlug sanitizes to lowercase alphanumeric-hyphen before writing |
| T-05-08 | Tampering | Negative or zero service price/duration | medium | mitigate | handleSvcPriceStep and handleSvcDurationStep validate parsed integer > 0 before DB insert |
| T-05-09 | Tampering | setWebhook called without prior deleteWebhook (re-registration conflict) | high | mitigate | handleActivate calls unregisterBotWebhook before registerBotWebhook unconditionally; documented STATE.md blocker |
| T-05-SC | Tampering | npm installs | low | accept | No new packages; crypto is Node.js built-in |
</threat_model>
