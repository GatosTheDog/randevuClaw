<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| booking agent → billing/queries.ts | Caller passes businessId + clientPhone; query layer enforces RLS via getConn() |
| DB transaction → memberships row | SELECT FOR UPDATE prevents concurrent reads during deduction |
| ledger insert → idempotency_key UNIQUE | DB-level dedup prevents webhook replay double-deductions |

## STRIDE Threat Register

| Threat ID | Category | Component | Severity | Disposition | Mitigation Plan |
|-----------|----------|-----------|----------|-------------|-----------------|
| T-08-03-01 | Tampering | insertBookingWithSessionDeduction concurrent race | high | mitigate | SELECT FOR UPDATE (.for('update')) inside db.transaction() acquires exclusive row lock before reading sessionsRemaining — two concurrent transactions serialize, only one deducts |
| T-08-03-02 | Tampering | cancelBookingWithRefund expiry bypass | high | mitigate | Expiry timestamp check (now > membership.expiresAt) performed INSIDE the transaction on the locked membership row — cannot be bypassed by a race between check and cancel |
| T-08-03-03 | Repudiation | ledger entry duplication on webhook replay | high | mitigate | idempotencyKey UNIQUE constraint on membershipLedger with .onConflictDoNothing() — duplicate deduction silently ignored; audit trail shows single entry |
| T-08-03-04 | Tampering | getEnforcementPolicy NULL → incorrect fallback | medium | mitigate | Explicit 'flag' fallback (policy === 'block' ? 'block' : 'flag') — any non-'block' value including NULL maps to safe 'flag' default |
| T-08-SC | Tampering | npm/pip/cargo installs | low | accept | No new packages required; all dependencies from Phase 7 |
</threat_model>
