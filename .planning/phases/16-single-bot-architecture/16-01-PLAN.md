  <name>Task 1: Add onboarding_completed to schema + generate migration with backfill</name>
  <files>src/database/schema.ts, src/database/migrations/0023_add_onboarding_completed.sql</files>
  <action>
    Add a new boolean column onboardingCompleted to the businesses pgTable definition in
    src/database/schema.ts. Place it after lastSessionThresholdCount (the last Phase 14
    column). Use the established nullable-then-set-default convention — this table is
    non-empty in production (same pattern as cancellationCutoffEnabled et al.) — but for
    this column use NOT NULL DEFAULT false since false is always safe for existing rows
    that have no session or an incomplete session; rows with a completed session will be
    updated by the backfill migration step below.

    Column declaration:
      onboardingCompleted: boolean('onboarding_completed').notNull().default(false),

    Add a comment above it matching the established pattern:
      // Phase 16 (ARCH-01): tracks whether the owner completed onboarding on this
      // business bot. When false, the business bot routes admin messages to the
      // onboarding state machine. Set to true by the terminal onboarding step handler.
      // Default false — safe for existing rows (backfill migration sets true where done).

    Also update the Business interface in src/database/queries.ts to include the new field:
      onboardingCompleted: boolean;

    Then create the migration file at src/database/migrations/0023_add_onboarding_completed.sql.
    The migration must:
    1. Add the column with the NOT NULL DEFAULT false constraint.
    2. Backfill: set onboarding_completed = true for any business that has a completed
       onboarding session (currentStep = 'done'). Use a correlated UPDATE with EXISTS so
       the backfill is atomic with the ALTER TABLE in a single transaction block.

    Write the migration as:
      BEGIN;
      ALTER TABLE businesses
        ADD COLUMN IF NOT EXISTS onboarding_completed boolean NOT NULL DEFAULT false;
      UPDATE businesses b
        SET onboarding_completed = true
        WHERE EXISTS (
          SELECT 1 FROM onboarding_sessions os
          WHERE os.business_id = b.id
            AND os.current_step = 'done'
        );
      COMMIT;

    Do NOT run the migration — the executor will run it with the project's standard
    migration command (npm run db:migrate or equivalent). Just write the file.
  </action>
  <verify>
    <automated>npx tsc --noEmit 2>&1 | head -30</automated>
  </verify>
  <done>
    src/database/schema.ts contains onboarding_completed column declaration with NOT NULL
    DEFAULT false. src/database/queries.ts Business interface includes onboardingCompleted:
    boolean. Migration file 0023_add_onboarding_completed.sql exists and contains both the
    ALTER TABLE and the backfill UPDATE. TypeScript compiles cleanly.
  </done>
