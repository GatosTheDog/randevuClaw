    FUNCTION 2 — handleSettingsToggle(action: string, business: Business, chatId: string): Promise<void>
    Dispatches the four binary toggle mutations. The action param is the menuAction value
    after the 'menu:settings:' prefix has been removed (e.g. 'slotless_on', 'cutoff_off').

    For each toggle, perform a direct DB update using withBusinessContext + db (appDb) via
    the pattern established in billing/queries.ts setEnforcementPolicy:
      await withBusinessContext(business.id, async () => {
        await db.update(businesses)
          .set({ <column>: <newValue> })
          .where(eq(businesses.id, business.id));
      });

    Then send a Greek confirmation message and re-show the settings menu with updated values.
    Because handleSettingsToggle only receives the original business object (pre-toggle), you
    must re-fetch the updated business after the mutation to pass to showSettingsMenu. Use
    findBusinessByOwnerTelegramId(chatId) to re-fetch (it was already imported in telegram.ts
    for the callback dispatch; import it here too from ../onboarding/queries).

    Toggle mapping:
      'slotless_on'      → set slotlessRequestsEnabled = true  → confirm: 'Τα αιτήματα χωρίς slot ενεργοποιήθηκαν.'
      'slotless_off'     → set slotlessRequestsEnabled = false → confirm: 'Τα αιτήματα χωρίς slot απενεργοποιήθηκαν.'
      'cutoff_on'        → set cancellationCutoffEnabled = true  → confirm: 'Η πολιτική ακύρωσης ενεργοποιήθηκε.'
      'cutoff_off'       → set cancellationCutoffEnabled = false → confirm: 'Η πολιτική ακύρωσης απενεργοποιήθηκε.'
      'multibooking_on'  → set allowMultiBooking = true  → confirm: 'Οι πολλαπλές κρατήσεις επιτρέπονται.'
      'multibooking_off' → set allowMultiBooking = false → confirm: 'Οι πολλαπλές κρατήσεις δεν επιτρέπονται.'
      'threshold_on'     → set lastSessionThresholdEnabled = true  → confirm: 'Η ειδοποίηση τελευταίου μαθήματος ενεργοποιήθηκε.'
      'threshold_off'    → set lastSessionThresholdEnabled = false → confirm: 'Η ειδοποίηση τελευταίου μαθήματος απενεργοποιήθηκε.'
      default            → send 'Άγνωστη ρύθμιση.' and return without DB change.

    Note on bookingMode: bookingMode is a two-value enum ('open_slots' | 'fixed_sessions').
    Do NOT add a toggle for it in this plan — it requires a more involved confirmation flow
    since changing it affects how the AI routes bookings. Route bookingMode changes to chat
    (already handled by the "γράψε στο chat" instruction in the settings display).

    FUNCTION — showTodaysAgenda(chatId: string, business: Business): Promise<void>

    Implementation steps (mirrors runAgendaSweep logic, WITHOUT claimAgendaSlot):
    1. Get today's Athens ISO date:
         const today = isoDateInAthens(new Date());
    2. Fetch bookings (confirmed + pending_owner_approval — same statuses as the agenda display):
         const bookingList = await listBookingsForDate(business.id, today, ['confirmed', 'pending_owner_approval']);
    3. Build service name map:
         const serviceNamesById = new Map<number, string>();
         for (const booking of bookingList) {
           if (!serviceNamesById.has(booking.serviceId)) {
             const service = await findServiceById(business.id, booking.serviceId);
             serviceNamesById.set(booking.serviceId, service?.name ?? 'Άγνωστη υπηρεσία');
           }
         }
    4. Format and send:
         const message = bookingList.length > 0
           ? formatAgendaMessage(bookingList, serviceNamesById)
           : 'Δεν υπάρχουν ραντεβού για σήμερα.';
         await sendTelegramMessage(chatId, message);
    5. After the agenda message, send the back button as a separate keyboard message:
         await sendTelegramMessageWithKeyboard(chatId, 'Τι άλλο θέλεις να κάνεις;', [
           [{ text: '« Πίσω στο Μενού', callback_data: 'menu:root' }],
         ]);

    CRITICAL: Do NOT call claimAgendaSlot or runAgendaSweep anywhere in this handler.
    The on-demand path must never consume the daily agenda slot (RESEARCH.md Pitfall 2).
