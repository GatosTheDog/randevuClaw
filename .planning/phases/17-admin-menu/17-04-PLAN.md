    FUNCTION 2 — showClientBalance(chatId: string, business: Business, relId: number): Promise<void>
    Shows the selected client's membership status and session balance.

    Implementation:
    1. const rel = await findClientBusinessRelationshipById(relId);
       Guard: if (!rel || rel.businessId !== business.id) {
         await sendTelegramMessage(chatId, 'Ο πελάτης δεν βρέθηκε.');
         return;
       }
       This ownership check (rel.businessId === business.id) prevents cross-tenant data reads
       even though business.id is derived from senderTelegramId (defense in depth).
    2. const clientPhone = rel.senderPhone;
    3. const membership = await getClientActiveMembership(business.id, clientPhone);
    4. Format message (Greek):
         const displayName = rel.clientName ?? clientPhone;
         If membership is null:
           `Πελάτης: ${displayName}\nΔεν υπάρχει ενεργή συνδρομή.`
         If membership.isUnlimited:
           `Πελάτης: ${displayName}\nΠακέτο: ${membership.packageName}\nΑπεριόριστες συνεδρίες\nΛήγει: ${membership.expiresAt.toLocaleDateString('el-GR')}`
         Otherwise:
           `Πελάτης: ${displayName}\nΠακέτο: ${membership.packageName}\nΥπόλοιπο: ${membership.sessionsRemaining} μαθήματα\nΛήγει: ${membership.expiresAt.toLocaleDateString('el-GR')}`
    5. Keyboard:
         If membership is not null:
           [[{ text: '📢 Αποστολή υπενθύμισης', callback_data: `menu:clients:nudge:${relId}` }]]
           followed by back button row
         If membership is null:
           Just back button row

    FUNCTION 3 — handleRenewalNudge(chatId: string, business: Business, relId: number): Promise<void>
    Sends a renewal reminder to the client via the business bot.

    Implementation:
    1. const rel = await findClientBusinessRelationshipById(relId);
       Guard: if (!rel || rel.businessId !== business.id) {
         await sendTelegramMessage(chatId, 'Ο πελάτης δεν βρέθηκε.');
         return;
       }
    2. const membership = await getClientActiveMembership(business.id, rel.senderPhone);
       If membership is null:
         await sendTelegramMessage(chatId, 'Δεν υπάρχει ενεργή συνδρομή — η υπενθύμιση δεν στάλθηκε.');
         return;
    3. Send renewal nudge to client via business bot:
         if (!business.botToken) {
           await sendTelegramMessage(chatId, 'Σφάλμα: δεν βρέθηκε το bot token της επιχείρησης.');
           return;
         }
         await botTokenStore.run(business.botToken, async () => {
           await sendTelegramMessage(
             rel.senderPhone,
             'Υπενθύμιση: Τα μαθήματά σας τελειώνουν σύντομα! Επικοινωνήστε για ανανέωση.'
           );
         });
    4. Confirm to owner:
         await sendTelegramMessage(chatId, `✅ Υπενθύμιση στάλθηκε στον ${rel.clientName ?? rel.senderPhone}.`);
    5. Send back button:
         await sendTelegramMessageWithKeyboard(chatId, 'Τι άλλο θέλεις να κάνεις;', [
           [{ text: '« Πίσω στο Μενού', callback_data: 'menu:root' }],
         ]);

    EXTEND handleMenuCallback — add cases. Note that menuAction values are:
      'clients'         → await showClientsList(chatId, business)
      'clients:balance' → id required → await showClientBalance(chatId, business, result.id!)

    Add the same id-undefined guard pattern used in Plan 17-03 for cases requiring an id.

<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| callback_data relId → findClientBusinessRelationshipById | relId is numeric, looked up then cross-checked rel.businessId === business.id |
| renewal nudge send → client Telegram chat | botTokenStore.run ensures correct per-business bot token; client senderPhone is resolved from DB row, not from callback_data |

## STRIDE Threat Register

| Threat ID | Category | Component | Severity | Disposition | Mitigation Plan |
|-----------|----------|-----------|----------|-------------|-----------------|
| T-17-14 | Tampering | Forged relId in menu:clients:balance callback targets foreign client | medium | mitigate | showClientBalance checks rel.businessId === business.id after lookup; foreign relId fails this ownership check and returns 'not found' |
| T-17-15 | Elevation of Privilege | Client sending menu:clients:nudge triggers unsolicited reminder to another client | medium | mitigate | handleCallbackQuery menu dispatch guards with findBusinessByOwnerTelegramId; non-owners never reach handleRenewalNudge |
| T-17-16 | Information Disclosure | Client balance data (package name, session count, expiry) exposed via crafted relId | medium | mitigate | rel.businessId === business.id cross-check; business.id derived from senderTelegramId not callback_data |
| T-17-17 | Denial of Service | Renewal nudge sent to wrong client via forged senderPhone | low | mitigate | senderPhone resolved from DB row (not from callback_data directly); DB lookup is ownership-checked before nudge send |
</threat_model>
