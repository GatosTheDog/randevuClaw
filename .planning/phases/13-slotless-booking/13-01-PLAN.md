--- insertSlotlessRequest ---
Signature: (params: { businessId: number; clientPhone: string; requestedSessionDate: string; requestedSessionTime: string; serviceId: number; idempotencyKey: string }) => Promise<SlotlessRequest | null>

Insert into slotlessRequests with status='pending'. Use .onConflictDoNothing().returning(). Return rows[0] ?? null (null = idempotent replay, caller ignores).

--- approveSlotlessRequest ---
Signature: (slotlessRequestId: number, businessId: number) => Promise<{ booking: Booking; request: SlotlessRequest } | null>

This is the atomicity-critical function (SLOT-03). Inside db.transaction(async (tx) => { ... }):

1. SELECT the slotless request row for UPDATE. Verify status='pending'. If not pending (already approved or rejected), return null.
2. Call getActiveMembershipForDeduction(businessId, clientPhone) using the tx connection. If null (membership lapsed), return null — the caller sends an error message to the owner.
3. Build a booking idempotency key: "slotless-approval:${slotlessRequestId}". Call insertBooking with the session date/time fields from the request, plus the idempotencyKey and a 2-hour expiresAt. Set bookingStatus='confirmed' directly (slotless-approved bookings skip the pending_owner_approval state — they are pre-approved by the owner's Ναι tap).
4. If insertBooking returns null (idempotent replay on same slotlessRequestId), look up the existing booking by requestId/clientPhone and use that row.
5. If membership.sessionsRemaining !== null and membership.sessionsRemaining > 0, call deductSession(membership.id, booking.id, "slotless:${slotlessRequestId}:deduction").
6. UPDATE slotlessRequests SET status='approved', bookingId=booking.id WHERE id=slotlessRequestId.
7. Return { booking, request: updatedRow }.

Use db.transaction() (admin db transaction, not getConn()) — the transaction wraps DB mutations across multiple tables. This matches how billing createMembership uses db.transaction in billing/queries.ts.

Note on insertBooking inside a transaction: insertBooking calls getConn() which reads from AsyncLocalStorage. Inside db.transaction, the transaction context is NOT automatically stored in currentTx AsyncLocalStorage — pass the transaction connection explicitly to insertBooking-equivalent inline SQL, or extract the insert inline within the transaction callback using `tx.insert(bookings).values(...)`. Use `tx` directly for all inserts/updates within the transaction to stay in the same transaction context.

Write the booking insert inline using tx rather than calling insertBooking (which uses getConn). The booking insert should:
- Set bookingStatus to 'confirmed' (not 'pending_owner_approval')
- Use .onConflictDoNothing().returning()
- Set calendarSyncStatus='pending', expiresAt=2h from now

Write the deduction inline using tx as well, mirroring the two-step approach in deductSession: tx.insert(membershipLedger).onConflictDoNothing(), then tx.update(memberships).set(sessionsRemaining - 1).

After the transaction, re-read the slotlessRequests row and return it with the booking.

--- rejectSlotlessRequest ---
Signature: (slotlessRequestId: number) => Promise<SlotlessRequest | null>

UPDATE slotlessRequests SET status='rejected' WHERE id=slotlessRequestId AND status='pending'. Use .returning(). Return rows[0] ?? null.

--- listSlotlessRequestsForClient ---
Signature: (businessId: number, clientPhone: string) => Promise<SlotlessRequest[]>

SELECT * FROM slotlessRequests WHERE businessId=businessId AND clientPhone=clientPhone ORDER BY createdAt DESC. Use getConn(). Return all rows.

--- countSlotlessRequestsSinceCheckin ---
Signature: (businessId: number, clientPhone: string, sinceDate: string) => Promise<number>

SELECT COUNT(*) FROM slotlessRequests WHERE businessId=businessId AND clientPhone=clientPhone AND createdAt >= sinceDate::date (cast the sinceDate ISO string to a date filter using gte and casting the createdAt timestamp). Use getConn(). Return Number(rows[0]?.count ?? 0).

The sinceDate is an ISO "YYYY-MM-DD" Athens local date string. Compare using: gte(slotlessRequests.createdAt, new Date(sinceDate + 'T00:00:00+02:00')) as a reasonable approximation for "since this Athens calendar date" (same approach as expiry comparisons in billing/queries.ts).
