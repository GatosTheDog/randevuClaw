    1. Fetch booking: const booking = await findBookingByIdUnscoped(bookingId). If null: send "Κράτηση δεν βρέθηκε." and return.
    2. Client ownership check: if booking.clientPhone !== senderTelegramId, log a warning and send "Δεν έχετε δικαίωμα ακύρωσης αυτής της κράτησης." and return. This is the cross-ownership guard per RESEARCH.md Security Considerations.

    4. Cancellation cutoff check: if business.cancellationCutoffEnabled and booking.calendarDate and booking.calendarTime exist, call hoursUntilSession. If hoursUntilSession < business.cancellationCutoffHours: send "Η ακύρωση δεν είναι δυνατή εντός " + business.cancellationCutoffHours + " ωρών από το μάθημα." and return. No DB mutation happens.
    5. Cancel: await updateBookingStatus(booking.id, 'cancelled').
    6. Credit restore: const membershipId = await findMembershipByBooking(booking.id). If membershipId !== null: await restoreCredit(membershipId, booking.id, booking:${booking.id}:credit).
    7. Calendar delete (best-effort, wrapped in try/catch): await deleteBookingFromCalendar(booking, business).
    8. Owner notification (best-effort, try/catch): if business.ownerTelegramId and business.botToken exist, use botTokenStore.run(business.botToken, async () => { await sendTelegramMessage(business.ownerTelegramId!, ownerText); }). ownerText: "Ακύρωση κράτησης από πελάτη:\nΗμερομηνία: " + booking.calendarDate + "\nΏρα: " + booking.calendarTime + "\nΠελάτης: " + booking.clientPhone.
