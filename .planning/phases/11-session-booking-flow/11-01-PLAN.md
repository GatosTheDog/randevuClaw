<objective>
Extend bookSessionInstance in src/session/manager.ts to atomically deduct 1 session credit from the client's active membership within the same DB transaction as the booking insert (SBOK-02). Phase 10 built the SELECT FOR UPDATE capacity guard and the booking insert; this plan adds the deductSession call inside the same transaction so credits cannot be charged without a booking, and bookings cannot be inserted without a credit charge.

Purpose: The atomic deduction is the correctness invariant that Phase 11 client-facing tools depend on. Without it, a client could book via the session flow without losing a credit.
Output: Updated src/session/manager.ts with deductSession wired into bookSessionInstance's transaction, TypeScript clean.

<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| booking insert → credit deduction | Both must land in the same transaction; rollback of either reverts both |
| activeMembership param → deductSession | Caller-supplied membership must belong to the same (businessId, clientPhone) — enforced by getActiveMembershipForDeduction's WHERE clause |

## STRIDE Threat Register

| Threat ID | Category | Component | Severity | Disposition | Mitigation Plan |
|-----------|----------|-----------|----------|-------------|-----------------|
| T-11-01 | Tampering | Double-deduction on idempotent replay of bookSessionInstance | high | mitigate | deductSession uses onConflictDoNothing on idempotency_key UNIQUE constraint; skip deduction call entirely on idempotent replay (bookingRows.length === 0) |
| T-11-02 | Tampering | Deduction without booking (partial transaction failure) | critical | mitigate | deductSession runs inside the same withBusinessContext transaction as the booking insert; any throw rolls back both atomically |
| T-11-03 | Elevation | Credit deduction for unlimited membership (sessionsRemaining IS NULL) | high | mitigate | Explicit null-check on sessionsRemaining before calling deductSession; matches billing/queries.ts CALLER RESPONSIBILITY comment |
| T-11-04 | Tampering | Cross-tenant membership deduction via crafted activeMembership param | high | mitigate | getActiveMembershipForDeduction includes businessId AND clientPhone in WHERE clause; caller-supplied activeMembership should only come from the same business context |
</threat_model>
