## Trust Boundaries

| Boundary | Description |
|----------|--------------|
| Gemini (external LLM) → function-executor.ts | Gemini-supplied tool-call sequencing and arguments cross into the DB-mutation layer; Gemini can request multiple mutating calls within one conversation turn |
| Conversation turn N → conversation turn N+1 (via Postgres conversationTurns.interactionId) | A malformed interactionId value persisted from one turn is trusted as `previous_interaction_id` input to the next Gemini API call for the same client |

| T-02-22 | Repudiation | function-executor.ts cancelAppointmentTool/rescheduleAppointmentTool (CR-03a/CR-03b) | medium | mitigate | Isolate Telegram notification failures in their own try/catch so a successful DB mutation is never reported back as a failure to the client/Gemini |
| T-02-23 | Denial of Service | ai-agent.ts aiBookingAgent while(true) loop (CR-01) | high | mitigate | MAX_TOOL_ROUNDS bound guarantees the webhook handler always returns, preventing an already-dedup-marked update_id from silently dropping the user's message forever |
