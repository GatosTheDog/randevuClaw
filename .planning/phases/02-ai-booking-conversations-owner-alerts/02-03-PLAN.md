## Phase Goal

**As a** client of a Greek service business, **I want to** check availability, book, cancel, reschedule, and ask questions about my appointment entirely through natural Greek conversation on Telegram, **so that** I can manage my booking with zero friction while the owner is reliably alerted to act on it and no two clients ever end up double-booked into the same slot.

> Note: ROADMAP.md's Phase 2 `**Goal**:` line is descriptive prose, reframed for Telegram per CONTEXT.md D-01/D-04 (see Plan 02-01's identical note for the full derivation). This plan delivers the "check availability" verb's underlying engine (BOOK-03) plus the Greek temporal-expression preprocessing every other booking verb depends on (D-13's 1-hour granularity, PITFALLS.md Pitfall 6's Greek date-parsing risk) — it runs in parallel with Plan 02-02 since neither touches the other's files, and both feed Plan 02-04's AI agent integration.

<objective>
Build the two standalone business-logic modules the AI agent (Plan 02-04) will call as tools: (1) `checkAvailability`, computing real 1-hour-granularity open slots for a service on a date, correctly accounting for each existing booking's own service duration and for closed days; (2) `resolveGreekTemporalExpressions`, pre-processing colloquial Greek date/time phrases into an unambiguous ISO hint appended to the message before it reaches Gemini, per RESEARCH.md's "pre-process before the LLM" recommendation. Both are pure, independently testable modules with no Telegram/Gemini dependency.

Purpose: RESEARCH.md's own pseudocode has two bugs this plan deliberately fixes: (a) the availability example assumes every existing booking has the SAME duration as the new request (wrong once services have distinct durations per D-12); (b) naive `new Date(...).getDay()` arithmetic on a UTC-hosted server does not reliably match "today in Athens." Both fixes are implemented here, once, so Plan 02-04 never has to re-derive them.
Output: `checkAvailability` and `resolveGreekTemporalExpressions`, both unit-tested against real scenarios (including a 20-phrase Greek temporal corpus).
</objective>

    In src/conversation/greek-preprocessor.ts, implement `resolveGreekTemporalExpressions(text, referenceDate)`: normalize a lowercased, diacritic-stripped copy of `text` via `stripGreekDiacritics` for matching (keep the ORIGINAL text for `annotatedText`, never mutate what the user actually typed). Relative-day keywords (match as whole-word substrings on the normalized copy): `σημερα`->0, `αυριο`->1, `μεθαυριο`->2, `σε (\d+) μερ(ες|ας)` regex ->N (capture group). Weekday keywords (diacritic-stripped, match both nominative and genitive by checking a `κυριακ`/`δευτερ`/`τριτ`/`τεταρτ`/`πεμπτ`/`παρασκευ`/`σαββατ` stem prefix so `Παρασκευή`/`παρασκευής`/`ΠΑΡΑΣΚΕΥΉ` all match): stem->target `weekdayOfIsoDate` value `κυριακ`->0, `δευτερ`->1, `τριτ`->2, `τεταρτ`->3, `πεμπτ`->4, `παρασκευ`->5, `σαββατ`->6. When a weekday stem is found: starting from `todayIso = isoDateInAthens(referenceDate)`, loop `offset` from 0 to 6, compute `candidate = addCalendarDays(todayIso, offset)`, and return the first `candidate` where `weekdayOfIsoDate(candidate) === target` — this single loop naturally implements "same day if today already matches" (offset 0 wins immediately) without a separate special case (Test 9). When a relative-day keyword is found instead, `resolvedDate = addCalendarDays(todayIso, N)`. If BOTH a weekday and a relative-day keyword somehow match, prefer the relative-day keyword (more specific). If neither matches, `resolvedDate = null`.
    For time: regex `/(\d{1,2})(?::(\d{2}))?\s*(π\.?\s?μ\.?|μ\.?\s?μ\.?)?/i` applied to the normalized text ONLY when preceded within ~15 characters by `στις` or immediately followed by a bare digit pattern matching an hour context (to avoid matching stray numbers like "3" in "σε 3 μέρες" as a time) — simplest robust rule: only attempt time extraction on a substring matched by `/στις\s+(\d{1,2})(?::(\d{2}))?\s*(π\.?\s?μ\.?|μ\.?\s?μ\.?)?/i` OR a bare leading `/^(\d{1,2})\s*(π\.?\s?μ\.?|μ\.?\s?μ\.?)?\b/i` match (handles Test 18/20's "time before weekday" word order and phrases without the literal word "στις"). Resolve hour/marker to 24h `"HH:MM"`: explicit `π.μ`/`πμ` (any spacing/dots) -> hour as-is (12->`"00:00"`); explicit `μ.μ`/`μμ` -> add 12 if hour &lt; 12; no explicit marker: if the text contains `πρωι` within the same phrase -> AM; if it contains `απογευμα`/`μεσημερι`/`βραδυ` -> PM; otherwise apply the bare-hour heuristic — hour 12 -> `"12:00"` (noon), hour 8-11 -> AM (`"08:00"`-`"11:00"`), hour 1-7 -> PM (add 12). Minutes default to `"00"` if not captured.

'άγνωστη'}]` `` to the ORIGINAL (non-normalized) `text`; if both are null, `annotatedText === text` unchanged (Test 17).
