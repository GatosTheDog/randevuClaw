## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| Sweep poller → Telegram API | sendTelegramMessage called inside botTokenStore.run(business.botToken, ...) — per-business token context enforced; no cross-business Telegram call possible |
| Sweep poller → Neon DB | findMembershipsExpiringIn7Days uses db (no RLS); filtered by businessId in WHERE — business isolation at application layer; no cross-business membership read |
| Neon DB → migrations script | Human applies 0008_expiry_notifications.sql; no automated push to avoid TTY UNIQUE constraint prompt |
| server.ts JEST_WORKER_ID guard | Prevents setInterval from keeping Jest alive; guard present on all existing pollers — new poller follows same pattern |

## STRIDE Threat Register

| Threat ID | Category | Component | Severity | Disposition | Mitigation Plan |
|-----------|----------|-----------|----------|-------------|-----------------|
| T-09-08 | Spoofing | runMembershipExpirySweep cross-business notification | high | mitigate | Outer loop iterates listAllBusinessIds(); findMembershipsExpiringIn7Days filters by businessId; botToken looked up per business — no message can be sent with wrong business token |
| T-09-09 | Information Disclosure | botToken in sweep | high | mitigate | botToken stored in business.botToken; only passed to botTokenStore.run() context; never passed to logger — mitigates T-09-09 pattern from Phase 4 |
| T-09-10 | Tampering | Concurrent sweep overlap duplicate sends | medium | mitigate | UNIQUE INDEX + onConflictDoNothing() atomically prevents duplicate dedup rows; insertMembershipExpiryNotification returns false on conflict so no second Telegram send occurs |
| T-09-11 | Denial of Service | Sweep holding Jest process alive | low | mitigate | JEST_WORKER_ID guard prevents startMembershipExpiryPoller() from being called in test environment; clearInterval pattern available for tests that import membership-expiry.ts directly |
| T-09-12 | Logic Error | Notification without DB migration applied | medium | mitigate | Human checkpoint gates server.ts registration on migration apply confirmation — poller is registered only after table is confirmed present |
