<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| owner → session tools | businessId FK chain (sessionCatalog → businesses → RLS) enforces ownership on all DB writes |
| concurrent clients → bookSessionInstance | Two concurrent clients on same instance — SELECT FOR UPDATE serializes the race at DB level |

## STRIDE Threat Register

| Threat ID | Category | Component | Severity | Disposition | Mitigation Plan |
|-----------|----------|-----------|----------|-------------|-----------------|
| T-10-02 | Tampering | Cross-tenant session creation (hacked businessId in manager.ts) | high | mitigate | WR-01: explicit businessId param on all mutations; withBusinessContext enforces RLS; subquery checks sessionCatalog.businessId in bookSessionInstance lock |
| T-10-03 | Elevation | Session capacity bypass (concurrent booking of last spot) | high | mitigate | SELECT FOR UPDATE on sessionInstances row inside getConn().transaction(); transaction serializes concurrent capacity checks; onConflictDoNothing on booking insert as last-line guard |
| T-10-07 | Denial of Service | Duplicate session instances from rrule expansion replay | medium | mitigate | onConflictDoNothing on batch insert; idempotencyKey UNIQUE constraint at DB level |
| T-10-08 | Information Disclosure | listSessions leaking cross-tenant session data | high | mitigate | getConn() is RLS-enforced; withBusinessContext sets app.current_business_id; WHERE businessId in sessionCatalog JOIN added as defense-in-depth |
| T-10-09 | Tampering | Invalid rrule string causing unhandled exception | medium | mitigate | RRule.parseString() throws on invalid RFC 5545 — try/catch wraps it, returns Greek error string via caller |
</threat_model>
