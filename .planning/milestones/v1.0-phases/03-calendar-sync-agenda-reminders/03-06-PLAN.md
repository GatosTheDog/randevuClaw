<objective>
Close the two BLOCKER gaps and two WARNING items found in the Phase 3 verification report (03-VERIFICATION.md score 5/7) so that OWNR-03 and NOTF-01 requirements are fully satisfied.

**BLOCKER 1 — 8am gate missing from agenda sweep (OWNR-03):** `runAgendaSweep()` fires on the first poller tick of the Athens calendar day at any hour. D-09 and the 03-04-PLAN.md must-have truth both require the send to be gated to Athens wall-clock time >= 08:00. The idempotency guard (`claimAgendaSlot`) must NOT be called during the pre-08:00 bailout.

**BLOCKER 2 — Wrong day label in 24h reminder (NOTF-01 / CR-01):** Line 145 of `src/scheduler/reminders.ts` hardcodes "αύριο" in the 24h reminder text regardless of whether `booking.calendarDate === todayIso` (a same-day appointment within the 24h window). The correct word is "σήμερα" for same-day and "αύριο" otherwise.

**WARNING 1 — Favicon requests crash the OAuth CLI (CR-02):** `scripts/setup-google-calendar.ts` runs the CSRF state check on every request, including browser-generated `/favicon.ico` and prefetch requests that arrive before the real OAuth callback. These carry no `state` param, so `receivedState !== state` is always true, causing the server to exit before the real callback arrives.

**WARNING 2 — googleRefreshToken missing from pino redact list (WR-02):** `src/utils/logger.ts` redact.paths does not include `googleRefreshToken`. A future `logger.info(business)` call would emit the OAuth token in plaintext.

Purpose: These four fixes complete Phase 3 and unblock the human Google Calendar verification checkpoint (03-03 deferred).
Output: Four modified files; two new test cases added to existing test suites; all 205+ tests continue to pass.
</objective>
