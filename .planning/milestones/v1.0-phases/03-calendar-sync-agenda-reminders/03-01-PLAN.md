## Phase Goal

**As a** business owner using RandevuClaw, **I want to** have every confirmed booking automatically reflected on my Google Calendar, get a daily Telegram summary of today's appointments, and have my clients automatically reminded before their visit, **so that** I never manually update a calendar or chase a no-show client, and my clients show up on time without me lifting a finger.

> Note: ROADMAP.md's Phase 3 `**Goal**:` line is descriptive prose, not the strict "As a / I want to / so that" format (same situation Plans 01-04/02-01/02-05 already documented for Phases 1-2 in this same project). This story is derived directly from that Goal plus Success Criteria 1-4, reframed for **Telegram, not WhatsApp** (CONTEXT.md D-01/D-02 — the Phase 2 messaging-channel pivot still stands, and SC3's "Meta-approved message template" clause does not apply this phase). No new scope is added: NOTF-01 (client reminders) is folded into the "so that" clause as the client-facing outcome the owner-facing story depends on, mirroring how Phase 2 folded OWNR-02 into its client story.

<objective>
Stand up the Phase 3 data substrate: 5 additive columns across `businesses` and `bookings` (Google Calendar OAuth token storage, calendar-sync status/retry tracking, and agenda/reminder sent-state idempotency guards), plus the full typed query layer every later Phase 3 plan (Calendar sync, Daily agenda, Reminders) imports without modifying `schema.ts` or `queries.ts` again — mirrors Plan 02-01's role in Phase 2.

Purpose: Getting the exact column shapes and the atomic "claim" query functions right here is what prevents the double-send/double-sync bugs RESEARCH.md's Pitfalls 3 (reminder idempotency bypass) and 5 (agenda sent multiple times) explicitly warn about. Every downstream plan's poller logic depends on these claim functions being genuinely atomic (UPDATE...WHERE...RETURNING), not a read-then-write check.
Output: Live Neon schema with 5 new columns, and a full typed query layer for calendar-sync/agenda/reminder state.
</objective>
