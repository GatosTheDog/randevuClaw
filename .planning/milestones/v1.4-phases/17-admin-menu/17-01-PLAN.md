      // Phase 17: admin menu callback pattern — menu:<action>[:<numericId>]
      // The menuAction discriminant is unique across all result types (RESEARCH.md Pitfall 1).
      const menuMatch = data?.match(/^menu:([\w:]+?)(?::(\d+))?$/);

<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| Telegram callback_query.data → parseCallbackData | Untrusted string from Telegram; regex-validated before any use |
| callback_data numeric ID → DB lookup | IDs from callback_data are numeric only; business ownership always re-derived from senderTelegramId |
| Owner command /menu → showAdminRootMenu | Only reachable after owner identity confirmed via ownerTelegramId match in handleFoundBusiness |

## STRIDE Threat Register

| Threat ID | Category | Component | Severity | Disposition | Mitigation Plan |
|-----------|----------|-----------|----------|-------------|-----------------|
| T-17-01 | Tampering | parseCallbackData — forged menu: callback_data | medium | mitigate | Regex validates shape; all DB lookups inside handleMenuCallback re-derive businessId from senderTelegramId via findBusinessByOwnerTelegramId (cross-tenant guard) |
| T-17-02 | Elevation of Privilege | /menu command sent by a client (not owner) | medium | mitigate | handleFoundBusiness checks ownerTelegramId === senderTelegramId before the /menu branch; clients never reach showAdminRootMenu |
| T-17-03 | Elevation of Privilege | menu: callback_query sent by a non-owner | medium | mitigate | handleCallbackQuery menu dispatch calls findBusinessByOwnerTelegramId; returns null for non-owners → ignores the callback |
| T-17-04 | Tampering | Stale keyboard buttons tapped after navigation | low | accept | Handlers are idempotent; clearing the old keyboard (editTelegramMessageReplyMarkup) reduces confusion but stale taps are harmless since ownership is re-checked |
| T-17-05 | Denial of Service | 64-byte callback_data overflow silently drops events | low | mitigate | assertCallbackDataSize logs a warning on overflow; all proposed callback_data strings verified at design time to be well under 64 bytes |
</threat_model>
