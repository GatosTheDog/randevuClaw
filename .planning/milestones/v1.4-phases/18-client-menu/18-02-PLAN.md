<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| cmenu:book:confirm:<instanceId> callback_data | instanceId extracted as Number() from regex; used only to query DB with business.id as co-key — cannot reach foreign tenant data |
| cmenu:book:yes:<instanceId> callback_data | Same; serviceId is re-fetched from DB at execute time, never from callback_data |

## STRIDE Threat Register

| Threat ID | Category | Component | Severity | Disposition | Mitigation Plan |
|-----------|----------|-----------|----------|-------------|-----------------|
| T-18-04 | Tampering | Forged instanceId in callback_data booking different business's session | medium | mitigate | bookSessionInstance receives business.id from the HMAC-verified webhook; session query joins to businessId column — foreign instanceId returns null/empty |
| T-18-05 | Tampering | Client sends cmenu:book:yes:<id> for a full session to attempt overbooking | medium | mitigate | bookSessionInstance uses SELECT FOR UPDATE capacity lock; concurrent bookings are serialised at DB level; full-capacity returns without creating a booking |
| T-18-06 | Elevation of Privilege | Client with no membership books a session in block-policy business | medium | mitigate | checkEnforcementAndGetMembership called first; block policy returns allowed=false before bookSessionInstance is ever called |
</threat_model>
